# Module 1 — Foundations and Prompt Design

## Overview

This is the entry point for all participants. No prerequisites are assumed beyond a general technical background. You do not need to have written AI code before to benefit from this module.

Module 1 establishes the mental model that every subsequent module builds on: how LLMs actually process input, why they behave the way they do, and how to structure prompts for reliable, repeatable results.

## Learning Objectives

By the end of this module, you will be able to:

- Explain how tokenisation, embeddings, and attention work at a conceptual level — well enough to predict and debug model behaviour
- Identify the four pillars of a well-formed prompt and apply them to a new task
- Use the persona pattern, XML delimiters, and Chain-of-Thought reasoning in a working prompt
- Implement a basic code review AI tool in Java (Spring AI), Python, or Node.js
- Explain why LLMs hallucinate and what prompt design choices reduce it

---

## Who Should Take This Module

**Everyone.** A developer who skips this module will write worse prompts. An architect who skips it will make worse design decisions. A manager who skips it will ask the wrong questions.

If you are an experienced developer who has already worked with LLM APIs, you may be able to move through sections 01 and 02 quickly, but section 03 (reasoning patterns) and the lab are worth doing in full.
